Kelola Akun Admin

Activity diagram alur penambahan, pengeditan, dan penghapusan data akun administrator.

💡 Rincian Arsitektur:

Hanya diakses oleh Super Admin / Admin berwenang. Menggunakan standard validator Django forms untuk email unik dan hashing password pbkdf2_sha256.

📋 Alur Aktivitas Berdasarkan Peran

Aktor (Admin) Sistem
1Membuka Menu Kelola Akun dan mengklik tombol "Tambah Akun" 2Menampilkan form Tambah Akun yang berisi input Username, Email, Password, Role, dan Polres terkait.
3Mengisi form secara lengkap dan mengklik tombol "Simpan" 4Menerima data, melakukan validasi format email, kekuatan password, dan keunikan username/email di database.
5(Menunggu perbaikan jika data input tidak valid) 6Jika input Tidak Valid, menampilkan pesan kesalahan/error di form. Jika Valid, meng-hash password dan menyimpannya ke database.
7(Melihat notifikasi sukses) 8Menampilkan pesan notifikasi berhasil menambahkan akun, memicu relasi profile otomatis (via Django signal), dan mengarahkan kembali ke daftar akun. Selesai.
💡 Rincian Arsitektur:

Mengambil data dengan primary key model user. Fitur mengubah role dan status aktif akun. Database melakukan update via QuerySet Django.

📋 Alur Aktivitas Berdasarkan Peran

Aktor (Admin) Sistem
1Membuka menu, memilih akun yang ingin diubah, dan mengklik tombol "Edit" 2Query data user terpilih ke database, lalu merender halaman form edit yang berisi data awal user tersebut.
3Mengubah nilai input (Email, Role, atau status Keaktifan) lalu mengklik "Update" 4Memvalidasi input baru. Memastikan email yang diubah tidak berbenturan dengan user lain.
5(Melihat error jika validasi gagal) 6Jika Valid, memperbarui record di database. Jika Tidak Valid, memunculkan pesan kesalahan.
7(Menerima pesan sukses) 8Menampilkan pesan notifikasi berhasil memperbarui akun, lalu me-redirect ke halaman utama kelola akun. Selesai.
💡 Rincian Arsitektur:

Keamanan integritas referensial. Menghapus akun admin akan mengeksekusi django `on_delete=models.CASCADE` pada profil, namun disarankan soft-delete (menonaktifkan status aktif) agar riwayat log data kecelakaan yang pernah diupload tidak hilang.

📋 Alur Aktivitas Berdasarkan Peran

Aktor (Admin) Sistem
1Memilih akun dari daftar dan mengklik tombol "Hapus" 2Menampilkan modal dialog konfirmasi konseptual: "Apakah Anda yakin ingin menghapus akun ini?"
3Memilih tombol Ya, Hapus (atau Batal jika ingin membatalkan) 4Mengevaluasi keputusan konfirmasi. Jika Batal, menutup modal dan kembali ke daftar tanpa mengubah database.
5(Menunggu proses database selesai) 6Jika konfirmasi disetujui, sistem menghapus baris akun dari tabel `coreapp_user` (atau mengubah properti `is_active` menjadi False).
7(Daftar diperbarui otomatis) 8Menampilkan pesan notifikasi berhasil menghapus data akun dan memperbarui list data akun. Selesai.

Polres bertindak sebagai entitas kewilayahan utama. Hanya Super Admin yang dapat menambahkan Polres baru dengan validasi field Kode Polres yang bersifat Unique.
