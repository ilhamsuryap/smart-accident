📖 Panduan Penggunaan Aplikasi

Panduan penggunaan Sistem Informasi Geografis pemetaan kerawanan kecelakaan lalu lintas.

1. Membuat Ruas Jalan

1

Masuk ke menu Pemetaan

Pilih menu Pemetaan pada sidebar untuk membuka halaman Kelola Ruas Jalan.

2

Tambah ruas jalan

Klik tombol Tambah Ruas Jalan, kemudian masukkan informasi ruas jalan.

3

Menggambar ruas jalan

Klik titik pada peta mengikuti jalur jalan hingga membentuk garis. Pastikan koordinat sesuai dengan kondisi jalan sebenarnya.

4

Isi Data Ruas Jalan

Data Ruas Jalan otomatis terisi dan klik simpan.

2. Mengunggah Data Kecelakaan

1

Masuk ke menu Pemetaan

Pilih menu Pemetaan pada sidebar untuk membuka halaman Kelola Data Pemetaan dan klik upload.

2

Download Template

Download Template format excel terlebih dahulu, dan sesuaikan format datanya untuk memudahkan dalam pengunggahan data kecelakaan, kemudian upload data.

⚠️ Catatan Penting

  • Jangan mengubah nama kolom pada template Excel.
  • Pastikan latitude dan longitude menggunakan format desimal (-7.7182, 111.5352).

3. Melihat Peta Kerawanan Kecelakaan

1

Membuka halaman peta kerawanan

Buka menu Pemetaan pada sidebar kemudian pilih tampilan hasil kerawanan untuk melihat segmen jalan berdasarkan tingkat risiko kecelakaan.

2

Memahami warna tingkat kerawanan

Setiap warna pada segmen jalan menunjukkan tingkat kerawanan kecelakaan.

🔴 Rawan Kecelakaan Sangat Tinggi
🟠 Rawan Kecelakaan Tinggi
🟡 Rawan Kecelakaan Sedang
🟢 Rawan Kecelakaan Rendah
🟢 Rawan Kecelakaan Sangat Rendah
3

Melihat informasi detail segmen

Klik salah satu ruas atau segmen jalan pada peta. Sistem akan menampilkan informasi seperti nama ruas jalan, jumlah kecelakaan, nilai Z-Score, dan tingkat kerawanan.

4. Mengelola Data Laka & Preprocessing

📂 Bagian 1: Data Laka Mentah (Automatly Report)

Langkah pengelolaan laporan kepolisian secara literal atau apa adanya dari berkas Excel kepolisian.

1

Halaman Utama Data Laka Mentah (Automatly Report)

Halaman ini menampilkan tabel seluruh data laporan kecelakaan mentah (laporan literal/apa adanya dari kepolisian). Di sini Anda dapat melihat ringkasan data, melakukan pencarian laporan tertentu, serta mengakses tombol fungsi utama seperti tambah data manual, import Excel, dan hapus data duplikat.

2

Input Manual Data Laka Mentah

Klik tombol Tambah Data Automatly Report pada halaman utama. Anda wajib memilih Polres dan Tahun kejadian, mengisi KET (contoh: "1 LR" untuk 1 luka ringan), serta menuliskan teks narasi mentah pada kolom No. Laporan, TKP & Koordinat, Kronologi, dan data korban.

3

Import Excel Data Laka Mentah

Klik tombol Import Excel pada halaman daftar. Pilih file Excel Anda, tentukan Polres dan Tahun, lalu klik unggah. Parser sistem akan mencari baris header kepolisian secara dinamis untuk diimpor.

4

Pembersihan Duplikat Data Laka Mentah

Jika terdeteksi adanya data dengan isi kolom utama yang sama persis, gunakan tombol Hapus Duplikat untuk membersihkan salinan data ganda secara otomatis dari database.

📊 Bagian 2: Manajemen Cluster Data (Kelola Data Preprocessing)

Mengelola data kecelakaan terstruktur yang siap dianalisis menggunakan algoritma K-Means dan AHC.

1

Halaman Utama Cluster Data (Kelola Data Preprocessing)

Halaman ini menyajikan tabel berisi seluruh data kecelakaan yang telah terstruktur dan bersih. Anda dapat memantau parameter-parameter penting (seperti Umur, TKP, Penyebab, Tanggal, Jam, Tipe Kendaraan, Kerugian Material) yang akan diproses lebih lanjut menggunakan algoritma klasterisasi.

2

Input Manual Cluster Data

Klik tombol Tambah Cluster Data pada halaman utama. Isi parameter terstruktur seperti Umur, TKP, Penyebab, Tanggal (nama hari otomatis disinkronkan oleh sistem), Jam, Jenis & Tipe Kendaraan, dan Kerugian Material.

3

Import Excel Cluster Data

Klik tombol Import Excel pada halaman daftar. Unggah file Excel dengan format kolom yang sesuai. Parser sistem mendukung pembacaan format penanggalan teks bahasa Indonesia (contoh: "1 Januari 2024").

4

Pembersihan Duplikat & Persiapan Analisis

Gunakan tombol Hapus Duplikat pada tabel Cluster Data untuk memastikan kebersihan data dari redundansi sebelum memulai pemrosesan algoritma K-Means atau AHC.

5. Analisis K-Means (Klasterisasi Hotspot Temporal/Waktu)

1

Unggah Data & Preprocessing K-Means

Fitur K-Means digunakan untuk memetakan slot waktu rawan kecelakaan berdasarkan kombinasi Jam dan Hari kejadian.

  • Sumber Data: Anda dapat mengunggah file Excel baru yang sesuai dengan mengklik tombol Download Template Excel terlebih dahulu, atau langsung mengklik tombol Gunakan Data Database untuk menggunakan seluruh data yang terdaftar di halaman Cluster Data.
  • Proses Preprocessing: Klik tombol Mulai Preprocessing. Sistem akan merangkum seluruh data kejadian kecelakaan menjadi 168 slot waktu mingguan (24 Jam x 7 Hari) dan menghitung akumulasi total kejadian di setiap jam.
2

Menjalankan Clustering

Setelah data selesai di-preprocess, klik tombol Proses Cluster untuk menjalankan algoritma clustering. Sistem akan memproses data untuk menemukan cluster-cluster waktu yang memiliki tingkat kerawanan laka lantas tertinggi.

3

Membaca Visualisasi Hasil K-Means

Klik Lihat Analisis Lengkap untuk membuka visualisasi terpadu:

  • Keterangan Pembagian Cluster:
    • 🔵 Cluster 1 - Rendah: Periode waktu dengan tingkat frekuensi kecelakaan terkecil (kondisi lalu lintas relatif aman).
    • 🟢 Cluster 2 - Sedang: Periode waktu dengan tingkat risiko menengah/rata-rata (memerlukan perhatian situasional).
    • 🔴 Cluster 3 - Tinggi: Periode waktu paling kritis dengan akumulasi kecelakaan tertinggi (prioritas utama penempatan patroli).
  • Grafik Akumulasi Tingkat Kerawanan (Bar Chart): Perbandingan total volume kecelakaan pada tiap kategori risiko, lengkap dengan interpretasi deskripsi otomatis.
  • Pemetaan Pola Waktu Kejadian (Scatter Plot): Pemetaan koordinat waktu 24 Jam (Sumbu X) vs Hari Senin-Minggu (Sumbu Y). Titik merah besar merepresentasikan slot waktu di klaster tinggi yang rawan.
  • Tren Temporal 24 Jam (Line Chart): Grafik fluktuasi kecelakaan per jam dengan garis bersegmen warna cluster untuk melihat lonjakan risiko pada jam-jam sibuk.
  • Metode Elbow (Elbow Method): Grafik justifikasi ilmiah yang membuktikan bahwa pembagian ke dalam 3 cluster ($k=3$) adalah pilihan yang paling optimal.
4

Merumuskan Rencana Patroli Taktis (Halaman Rekomendasi AI)

Di bagian bawah halaman hasil K-Means, klik tombol Rumuskan Langkah Penanganan & Pencegahan Terpadu. Sistem akan menghubungkan hasil klaster ke AI Gemini untuk merumuskan usulan strategis:

  • Jadwal Patroli Taktis: AI akan menyusun jadwal patroli lengkap (Hari, Jam, Fokus Pengawasan, dan Unit Penanggung Jawab) yang diarahkan tepat ke slot waktu klaster tinggi (merah).
  • Tindakan Infrastruktur: Usulan perbaikan lampu jalan, marka jalan, atau pemasangan rambu di area rawan.
  • KPI Target & Program Kerja: Program aksi jangka pendek (1 bulan), menengah (6-12 bulan), serta target persentase pengurangan kecelakaan.

6. Analisis AHC (Klasterisasi Karakteristik & Faktor Risiko)

1

Unggah Data & Preprocessing AHC

Sama seperti K-Means, AHC membutuhkan data yang bersih. Unggah file Excel data preprosesing baru atau klik Gunakan Data Database di menu AHC, lalu tekan tombol Mulai Preprocessing untuk memuat dan menyusun karakteristik kasus.

2

Menentukan Jumlah Klaster & Menjalankan AHC

Fitur AHC (Agglomerative Hierarchical Clustering) digunakan untuk mengelompokkan kecelakaan berdasarkan karakteristik pelaku (usia), faktor penyebab (kelalaian, pelanggaran, teknis), serta dimensi waktu & hari.

  • Menentukan Jumlah Klaster: Masukkan angka jumlah klaster yang diinginkan pada kolom input (minimal 2, maksimal 10) lalu klik Proses Clustering.
3

Membaca Visualisasi Karakteristik & Eksplanasi AI

Klik tombol Lihat Analisis Lengkap setelah pemrosesan selesai untuk membuka halaman grafik lanjutan:

  • Tabel Karakteristik Klaster: Merangkum karakteristik utama setiap klaster (populasi kasus, rata-rata umur korban, faktor penyebab dominan, sesi waktu dominan, dan hari dominan).
  • Visualisasi Sebaran Klaster (PCA): Proyeksi spasial data 2D yang menunjukkan kerenggangan atau kedekatan karakteristik antar klaster.
  • Struktur Hierarki Klaster (Dendrogram): Diagram pohon pengelompokan hierarkis dengan garis cut-off merah sebagai batas pemisah jumlah klaster terpilih.
  • Boxplot Umur per Klaster: Grafik distribusi untuk melihat rentang sebaran usia pengendara di setiap klaster secara visual.
  • Analisis Faktor Berdasarkan Kelompok Umur: Stacked bar chart yang menghubungkan kelompok usia pengendara dengan kontribusi faktor penyebab kecelakaan (Kelalaian, Pelanggaran, Teknis).
  • Eksplanasi AI Otomatis: Sistem otomatis memanggil Gemini AI untuk merumuskan narasi ilmiah (interpretasi data) untuk kelima grafik di atas. Anda dapat melihat hasil narasinya di dalam kotak biru berlabel "Generate by AI" di bawah masing-masing chart.
4

Merumuskan Strategi Kebijakan Taktis (Halaman Rekomendasi AHC)

Di bagian bawah halaman hasil AHC, klik tombol Rumuskan Langkah Penanganan & Strategi Terukur. Sistem akan membuka halaman rekomendasi kebijakan AHC khusus yang dirumuskan oleh AI Gemini:

  • Sintesis Risiko: AI akan menghubungkan variabel faktor dominan global dengan kelompok umur pengendara yang paling rentan untuk menyusun ringkasan prioritas penanganan.
  • Matriks Intervensi: Usulan tindakan pengawasan spesifik (seperti edukasi ke sekolah/kampus untuk target pengendara muda) dan program perbaikan fasilitas jalan.
  • Jadwal Pos Pengawasan & Program Kerja: Jadwal pos penjagaan di jam-jam krusial serta program aksi jangka pendek dan menengah yang terukur.
